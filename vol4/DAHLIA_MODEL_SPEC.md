# 🌸 大丽花 GLB 模型规范文档

> 供建模师在 Blender / C4D / Maya 中制作，导出 GLB 后由前端加载并驱动动画与材质。

---

## 一、整体要求

| 项目 | 要求 |
|------|------|
| 格式 | `.glb`（二进制 glTF） |
| 坐标系 | Y 轴朝上（Blender 默认导出即可） |
| 单位 | 1 单位 = 1 米（花朵整体直径约 0.6~1.0） |
| 原点 | 花朵中心（花心顶部）放在世界原点 `(0, 0, 0)` |
| 朝向 | 花朵正面朝 Y+ 方向（向上生长） |
| 面数 | 单片花瓣 200~600 面，总计控制在 **5万面以内** |
| 材质 | 不需要贴图，导出时可以不带材质（前端会用自定义 Shader 替换） |

## 二、层级结构（⚠️ 重要）

```
Dahlia_Root              ← 根空物体 (Empty)
├── Core                 ← 花心/花蕊（可选，一个 mesh）
├── Layer_0              ← 第0层（最内层）空物体
│   ├── Petal_0_0        ← 第0层 第0片花瓣 mesh
│   ├── Petal_0_1
│   ├── Petal_0_2
│   └── ...
├── Layer_1              ← 第1层 空物体
│   ├── Petal_1_0
│   ├── Petal_1_1
│   └── ...
├── Layer_2
│   └── ...
├── ...
├── Layer_7              ← 最外层
│   └── ...
└── Stem                 ← 花茎（可选，一个 mesh）
```

### 命名规则

| 名称 | 格式 | 说明 |
|------|------|------|
| 根节点 | `Dahlia_Root` | 整朵花的父级 |
| 花心 | `Core` | 可选，花蕊部分 |
| 层级组 | `Layer_{N}` | N 从 0 开始，0=最内层 |
| 花瓣 | `Petal_{层号}_{序号}` | 如 `Petal_2_5` = 第2层第5片 |
| 花茎 | `Stem` | 可选 |

## 三、花瓣建模要点

### 3.1 每片花瓣的轴心点（Origin）

> **⚠️ 这是最关键的一点！**

每片花瓣的 **轴心点（Origin / Pivot）必须设在花瓣基部**（连接花心的那一端），而不是花瓣中心。

```
       🌿 尖端
      ╱    ╲
     ╱      ╲
    ╱        ╲
   ╱          ╲
  ╱            ╲
 ╱              ╲
⬤ ← 轴心点在这里（基部）
```

**原因：** 前端会围绕这个轴心点做旋转动画（张开/合拢），轴心在基部才能实现自然的展开效果。

### 3.2 花瓣初始姿态

- 每片花瓣建模时保持 **"合拢"状态**（花苞姿态）
- 即花瓣基本竖直，尖端朝上或略微内收
- 前端代码会通过旋转驱动花瓣展开到最终姿态

> 如果你更习惯建"展开"状态的花也可以，但需要告诉我，我反向处理。

### 3.3 花瓣形态建议

| 层级 | 花瓣数 | 大小 | 形态特征 |
|------|--------|------|----------|
| Layer_0 | 5~6片 | 最小 | 紧密包裹花心，窄长 |
| Layer_1 | 7~8片 | 小 | 略微展开 |
| Layer_2 | 9~10片 | 中小 | 开始有弧度 |
| Layer_3 | 11~12片 | 中 | 明显碗状弧面 |
| Layer_4 | 13~14片 | 中大 | 宽圆，边缘自然 |
| Layer_5 | 14~15片 | 大 | 大幅展开 |
| Layer_6 | 15~16片 | 较大 | 近水平 |
| Layer_7 | 15~16片 | 最大 | 最外层，可微微下垂 |

### 3.4 花瓣造型细节

- ✅ 边缘可以有**不规则起伏**（不要太光滑的椭圆）
- ✅ 花瓣有**纵向弧度**（从基部到尖端是弧线，不是直板）
- ✅ 花瓣有**横向碗状弧面**（横截面是弧形，不是平片）
- ✅ 尖端可以微微**向内卷或向后翻**
- ✅ 每片花瓣之间有**角度错开**（不要完全均匀对称，加一点随机）
- ✅ 内外层之间角度错开（相邻层交错半个花瓣位）

## 四、几何要求

### 4.1 法线

- 每片花瓣需要有**正确的法线方向**
- 推荐使用 **柔和法线（Smooth Shading）**
- 如果用 Blender：`右键 → Shade Smooth`，然后加 Auto Smooth（30°~45°）

### 4.2 UV

- 每片花瓣需要 UV 展开
- 推荐简单的 **从上方投影** 或 **沿花瓣长度方向展开**
- UV 的 U 方向 = 花瓣长度方向（0=基部，1=尖端）
- UV 的 V 方向 = 花瓣宽度方向（0~1）
- 前端 Shader 会依赖 UV 做颜色渐变和透明度衰减

```
U=0 (基部) ──────────────→ U=1 (尖端)
V=0 ┬
     │     花瓣面
V=1 ┴
```

### 4.3 面数参考

| 部分 | 单个面数 | 数量 | 小计 |
|------|----------|------|------|
| 花瓣 | 300~500 | ~100片 | 30k~50k |
| 花心 | 500~1000 | 1个 | 1k |
| 花茎 | 200~500 | 1个 | 0.5k |
| **合计** | | | **~50k 以内** |

## 五、导出设置（Blender）

### 5.1 导出路径

`文件 → 导出 → glTF 2.0 (.glb)`

### 5.2 导出选项

| 设置项 | 值 |
|--------|-----|
| Format | glTF Binary (.glb) |
| Include → Selected Objects | ❌ 不勾（导出全部） |
| Transform → +Y Up | ✅ 勾选 |
| Mesh → Apply Modifiers | ✅ 勾选 |
| Mesh → UVs | ✅ 勾选 |
| Mesh → Normals | ✅ 勾选 |
| Mesh → Vertex Colors | ❌ 不需要 |
| Material | 可以不带（None），或者带个简单的 |
| Animation | ❌ 不需要（动画由前端代码驱动） |
| Compression | ❌ 不勾（避免兼容性问题） |

## 六、自检清单

导出前请逐项确认：

- [ ] 根节点命名为 `Dahlia_Root`
- [ ] 花瓣按 `Layer_N / Petal_N_M` 结构组织
- [ ] 每片花瓣的轴心点在**基部**（不是几何中心）
- [ ] 花瓣初始为合拢/半合拢姿态
- [ ] 所有花瓣有 Smooth Shading
- [ ] 所有花瓣有 UV 展开（U=长度方向）
- [ ] 总面数 < 5万
- [ ] 导出时 Apply 了所有 Modifier
- [ ] 没有多余的隐藏物体/相机/灯光
- [ ] GLB 文件大小 < 5MB

## 七、可选增强

以下不是必须的，但如果能做会更好：

1. **厚度信息**：花瓣如果有厚度（Solidify），可以保留，Shader 会用来做 SSS
2. **顶点色**：可以用顶点色标记花瓣基部(黑)→尖端(白)的渐变，作为 Shader 的额外输入
3. **花心细节**：花心可以建一些小的花蕊柱状体，增加细节
4. **多个 Shape Key**（形变键）：如果建模师愿意做，可以建 2 个 Shape Key：
   - `Closed`（合拢状态）
   - `Open`（完全展开状态）
   - 前端可以通过 morph target 做更自然的展开过渡
